闲侃

Veröffentlicht am 2017-12-21
  • 一晃一年又8天，经历了两家公司，第二家公司忙一些。忙的过程中，我发现同事都在电脑前做了一个便签，有的是纸，有的是软件，感觉挺方便，把要做的记下来，可以腾出脑筋做事，而不是想我需要做什么。
  • 现在可以说算是童老师口中的coder了。
  • 最近中兴的跳楼案引发了程序员圈子的中年危机的讨论，我个人还没什么感觉，虽然中年危机一般是年轻人才有.
  • 我没有想过自己能做的很强，比如做到CTO之类的，但是还是在顺应自己的感觉，比如自己喜欢的方式和内容。
  • 我想大多人还是很普通的，不可能人人都是架构师，这些人我想到最后总有一条出路，当然我希望最好到了30，40还能coding，不写代码我感觉没啥意思，当然只是现在的想法。
  • 最近看到360的老总周总的传闻，他曾经也是一个代码高手，在公司写了一款给同事用的工具软件一炮打响了自己。同时还有张小龙，每当他的同事说他的idea无法实现时，他便会抛一段自己写代码给同事，说这样实现可以吧。虽说有人天生聪颖，毅力过人，但是这些事情听到之后很激励人，就像现在我们公司准备让我们做的前端宣传网站。
  • 关于前端宣传网站，是个node做api层的网站，能拓展一下node的技术栈是这个东西对我来说最大的吸引力，在上一家公司，后台是node，然而并没有很多的去写这个，这比普通的靠书学习的方式更加有压迫力。
  • 关于工作，现在只能吐槽一句，尽力而为，事还是要一件一件的做，把事罗列在便签上，尽量提高效率，不可能一口吃个胖子。
  • 关于游戏，现在除了很闲的时候才会吃鸡一把，和同事一起吃鸡是件很有意思的事，输赢不重要，重在乐在其中。
  • 关于学习，我现在觉得还是不能把兴趣磨没了，找点感兴趣的技术文章看，投其所好，比死磕一本技术书籍好，最好是两者结合。
  • 沉默是金勿忘我，百事从师悟需己

道别

Veröffentlicht am 2017-12-09

​ 几年前，我还在上大学，忙于课业，我有幸认识了王静逸哥哥，有天他到我姐姐家，我也在那里，于是我们开始了最有意思的话题，计算机。由于当时还是个小菜鸡（虽然现在也是），所以问的最多的是他工作的话题。
​ 他很有个性，要我每天看两小时的书，我说你每天都看两个小时吗，他呵呵一笑，然后告诉我他看书不分昼夜，当时的我惊讶无比，想了想自己可能做不到，包括现在都没有做到。
​ 他说程序员到后面就是拼手速了，看谁打得快了。我说不会被一个问题卡住吗。他又笑了笑，说他在他的领域现在已经没有解决不了的问题了。我也大笑，好酷啊。
​ 他是游戏前端，谈到游戏，他说做游戏比玩游戏要有趣的多，我是一个很喜欢打游戏的人，对于做游戏比游戏还要好玩，有点不理解。当他打开visual studio调试qq炫舞的代码时，那快捷键的飘逸按法，我似乎有点感觉了。然而我现在是一个web前端，与他相比差了一大截，可我也很自豪，因为我也是一个程序狗，一只很极客的小年轻。

​ 后来他创建了神兔游戏，去追寻他自己的理想。他一直都跟我说做游戏很赚钱，确实，一个游戏只要有人玩，就会产生一些消费，而这是一笔很可观的收入，所以游戏公司的待遇自然会很高。当然，他也是很辛苦的，经常听他说，在开着的电视机前一坐就是一夜晚（写代码）。

​ 我其实还想和他聊聊，然而现在已经不行了，他与我姐已经分道扬镳，再讲话可能会很尴尬，即使我不觉得。我总是没有这些所谓的常识，呵呵。

​ 现在我有幸接触到了webgl，作为一个html5的实践工作者，这个东西似乎把我跟王静逸哥哥的东西连接到了一起，以前他要我学图形学，当时让他失望了，现在我可以学。但是不是为了他，而是为了自己，为了有趣，为了满足自己的极客感。

​ 好了，再见，王静逸哥哥，祝你回汉以后能顺风顺水，更上一层楼！

JS对象漫谈

Veröffentlicht am 2017-11-26

如何转为number

  • Number(‘1’) === 1
  • parseInt(‘1’, 10) === 1
  • parseFloat(‘1.23’) === 1.23
  • ‘1’ - 0
  • +‘1’

全局对象window

  • 浏览器在加载时首先会创建window对象
  • window的属性分为两类： ECMAScript规定的和浏览器私有的（alert，prompt，confirm，console, history, document等）
  • document属于dom规范范畴 ,history属于bom规范范畴

详解Number

  • Number(‘1’) 类型转换
  • var n = new Number(1) 1 => {valueOf(): 1,..},通过valueOf获取到原始值1
  • 使用直接量赋值 var n = 1;在早期是没有toString方法的，而现在直接使用toString会临时转换为Number对象包装后的“1”然后使用toString，最后把这个包装对象丢掉（磨砂次）,这就是为什么给n.xxx = 1后，然后取n.xxx是undefined的原因

parseInt 与 toString进制问题

  • parseInt第二个形参传入一个进制，代表将第一个实参是什么进制的，然后转化为10进制输出
  • toString可以传入一个参数代表将要转化的进制，把调用者转化为对应进制的字符串

slice

  • 为什么叫slice？外国人切面包叫slice，js slice可以“切”数组，字符串。

5个falsy值

  • 0
  • NaN
  • ‘’
  • null
  • undefined
  • false

所有对象都是真值

原型

var n1 = new Number(1)

n1.proto === Number.prototype

n1.proto.proto === Object.prototype

proto 和 prototype

s.proto 也是String的公用属性的引用

对象.proto === 函数.prototype 它们的值是 [Prototype]

var obj = 函数.prototype

obj.proto === Object.prototype

var obj2 = 函数

obj2.proto === Function.prototype

Function.proto === Function.prototype

Function,prototype.proto === Object.prototype

是不是治好了你的颈椎病？（逃）

人工智能强势登场

记得会场上IEEE Fellow蔡自兴院士说了一句让我印象十分深刻话，他说年轻人不要偷懒哦，小心机器人取代你。对啊，随着人工智能的发展，计算机从计算到感知到认知，可以想象在将来的某一天，机器人也有情感，它将不再是一堆冰冷的钢铁，而是一个有趣的小伙或是一个小姑娘。

上面这幅图讲述了人工智能可能会替代的职业，从电话推销员到政府职员，涉及面覆盖的很广，我想这只是其中的一部分而不是全部。可见人工智能的威力。

why AI？

互联网行业是近几年的热门，非计算机专业的人都想来这里捞一桶金，然而随着发展，互联网的流量能挣到的钱正在消失，维护的成本在日益增加。人工智能的处理代替传统人力工作，从一定程度上降低的这方面的成本，所以这是为什么目前国家在大力开展人工智能的建设的原因之一。

简单的AI思想

现在有一个场景，国外有些职场人士每天喜欢喝热的咖啡，他们需要看咖啡机里的咖啡是否是满的，是否是热的，是否已经空了等等情况来做出加热，加水等操作，如果把这个交给人工智能来做，应该怎么做呢？

会上讲到一种思路是使用value function

on off
full -2 -5

value function

左侧代表咖啡机是否装满了，右侧代表是否开启加热功能时的状态，用这种映射关系来指挥计算机。当函数的自变量为-2时，会得到一个状态：咖啡机满了且需要加热。

个人的奇思妙想

如果机器人有了情感他会听人类的吗？

比方说现在用生物技术做出了一种生物，你觉得你能很好的控制它么？

这就像当老师管学生一样，老师说的大道理再多，学生会去执行吗？

同理你每天命令有感情的计算机工作，它会不会厌烦了。

当然，这只是个人拙见，还待科学家解释。
